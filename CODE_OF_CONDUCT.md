# Code of Conduct

## Our Pledge

We are committed to making this project a welcoming, respectful, and harassment-free environment for everyone, regardless of age, body size, disability, ethnicity, gender identity or expression, level of experience, nationality, personal appearance, race, religion, sexual identity or orientation, or any other personal characteristic.

We expect all participants to help create a positive and productive community.

## Expected Behavior

Participants are expected to:

- Be respectful and considerate of others.
- Communicate with empathy and professionalism.
- Give and accept constructive feedback gracefully.
- Focus on what is best for the project and its community.
- Respect differing viewpoints and experiences.
- Assume good intent when possible, while still taking responsibility for impact.

## Unacceptable Behavior

Unacceptable behavior includes, but is not limited to:

- Harassment, intimidation, threats, or personal attacks.
- Discriminatory language or behavior.
- Sexualized language, imagery, or unwanted sexual attention.
- Trolling, insulting comments, or deliberate disruption.
- Publishing private information without explicit permission.
- Repeatedly contacting someone after being asked to stop.
- Any behavior that would reasonably be considered inappropriate in a professional setting.

## Scope

This Code of Conduct applies within all project spaces, including but not limited to:

- GitHub issues, pull requests, discussions, and comments.
- Project chat rooms, forums, mailing lists, and social media.
- Conferences, meetups, or other events related to the project.

It also applies when an individual is representing the project in public spaces.

## Enforcement

Project maintainers are responsible for clarifying and enforcing standards of acceptable behavior. Maintainers may take appropriate action in response to behavior that violates this Code of Conduct.

Enforcement actions may include:

- A private or public warning.
- Temporary or permanent removal from project discussions.
- Temporary or permanent blocking from the project repository or organization.
- Rejection or removal of contributions.
- Reporting behavior to relevant platforms or authorities when necessary.

Maintainers should apply enforcement fairly and consistently.

## Reporting

If you experience or witness unacceptable behavior, please report it to the project maintainers.

sean@headzoo.io

## Attribution

This Code of Conduct is adapted from common open source community standards and is intended to support a respectful, inclusive, and productive project environment.
